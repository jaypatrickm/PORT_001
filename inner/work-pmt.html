Work - PetSmart Petboard Touch Prototype

A touchscreen interface built with html, css, javascript, ajax and jquery.

Programmer

September 2014

The container full of moving containers!! Putting everything together was challenging. It contained a moving container of divs that each had their own sets of movement.

This was the first time I created something with so much moving animation. I was concerned with the amount of processing and loading everything would create because each inner container is an iframe of a new set of html. My experience at this point taught me to minimize the amount of code as much as possible and keep my code clear and organized because I was not the only person that would be working on it. I also used Velocity JS in some applications to improve speed.

My team had previously created the grid container sizes and where they would move. I created the html base, css animations for each movement and the timing through javascript.

Every inner container is its own html, each exists as their own application inside the Insteo database. The applications are tied to an account within Insteo's CMS. You log in and customize the different parts of the layout; you could change colors, text, and images.

Each div was an application I built with its own html, css, and javascript files. I built the html and javascript either from scratch or using a plugin that my team had found to do what the client wanted. The javascripts contained code that would make an ajax call to the database and retrieve the custom infomation and send it over back as JSON data. I then used jquery to apply the new custom attributes to the original html and css.

I programmed the javascript.
