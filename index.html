QUEEN MARY WAYFINDING & TIMELINE

HTML5, CSS3, JavaScript, jQuery

ROLES Programmer, Designer

DATE May 2013

DESCRIPTION A luxurious cruise ship turned hotel permanently docked in Long Beach, CA., the Queen Mary Ship asked Insteo to build a touchscreen kiosk to show a map, timeline, various events, stores and restaurants.

CHALLENGES Build for a full HD 1080 x 1920 touchscreen. Incorporate multiple web applications that are separately built inside Insteo CMS system. Ensure you cannot notice that it is a website. Implement concerns for security: not able to surf the web, no reaching outside websites, etc. One month timeframe. I am not working with any other designers or programmers.

SOLUTIONS I did not have any previous JavaScript experience before this job and because I was working solo, my main concern was giving them a timeline that looked appropriate and functioned well. I achieved this by incorporating touchSwipe and scrollTo plugins, both of which I had no experience with beforehand.

The design of the timeline reflected the birch wood interior of the ship as well as a brass metal I saw in pictures of the ships control room.

For security, I hid the cursor and disabled any kind of double-clicking events. It took some on-site testing to really discover if there were any ways to start freely browsing the browser.

Ultimately, I was very happy with what I was able to accomplish having been my first big project at Insteo. The project won Insteo a Gold Apex Award at the Digital Signage Expo 2014.

OZIO TOUCH

HTML5, CSS3, JavaScript, jQuery

ROLES Programmer, Designer

DATE June 2014

DESCRIPTION A multi-functional entertainment pad paired with pay-per-minute massage chairs placed throughout public malls. Insteo worked with Ozio to create a touchscreen interface to accompany their line of massage chairs.

CHALLENGES Build for a small touchscreen, exact width unknown. Must have sections for Music, Movies, News, Surveys and Deals. Show different massage times for sale. Show a timer. Show and play videos and music. Have a built in survey. Create QR code for deals. I would not be working with any other designer or programmers. One Month timeframe.

SOLUTIONS The client did not have an exact idea in mind and simply wanted something to show to his clients to sell his chairs. Since he was targeting public malls, I wanted to give him a modern looking touchscreen. After my research, I implemented a flat design with soft colors to give the user something nice and easy to use. Since they would be in a massage chair, I wanted to give them an option for different themes to change the background.

The most difficult part was implementing a video and music player which I achieved with a JPlayer plugin. To give it more of a touch screen feel, I used CSS Animations to create all of the transitions that happen when you touch on any of the links.

I think the project turned out really nice because of the soft colors mixed with the smooth transitions.

LOOP ANIMATED AD

HTML5, CSS3, JavaScript, jQuery

ROLES Programmer, Designer

DATE April 2013

DESCRIPTION A customizable, eye-catching, animated ad displaying in-store sales. Insteo creates digital signs for companies to use for their own products. I was tasked with creating an animated display for the convenience store Loop.

CHALLENGES Build for a full HD 1080 x 1920 vertical screen. It will display multiple convenience store products. Must coincide with the Loop Brand. Must animate each product with a price, name, and small tagline. It should be easy to add products and customize the look. Two week deadline. No other designers or programmers.

SOLUTIONS My main concern with this project was ease of use. I thought about what was the easiest way to add in a product. So I designed the template to use an image of the product that was dead center in a 1080 x 1920 frame. All the user would need is a product with a transparent background or in this case a white background. Then they would just need to upload the product, name, price, and tagline.

I used jQuery to animate all of the different items and customize the colors. Animating was the most difficult part. It was my first time using jQuery animate and I was learning to chain my animations and timing them to show the product for the correct amount of time.

Although the project looks very simple, I think that's what makes it really successful. It is easy to use and easy to customize. Insteo was actually able to use this for multiple clients by just changing the background and colors, you would just need to change the product image feed. You could show fashion products, car products, anything! The project is built in landscape and is used on a rotated vertical tv monitor.

PetSmart PetBoard Touch

HTML5, CSS3, JavaScript, jQuery

ROLES Programmer

DATE September 2014

DESCRIPTION Insteo worked with a PetSmart branch to create an in-store ipad touchscreen application to help generate more interest in their pet adoptions, events, social media, and store sales.

CHALLENGES Build for an Ipad. Use existing Insteo applications. Have a moving dashboard. Add Social Media. One month deadline. Work with their designers and our Insteo team

SOLUTIONS The difficult part of this project was coding the moving parts to move smoothly. It was difficult because the top level divs that were moving, were all made of iframes of different applications that each had their own set of code. So I had to be very efficient with my code. Some of the applications still needed to be built and designed. Receiving the okays back from the client, as well as redesigns, created a shorter timeframe to code everything together.

The movement is created using CSS transitions. My team provided help creating each div application with jQuery plugins that I needed to dig into and adjust to make them appropriately customizable within the Insteo database. The social media is also connected to the Insteo database and shows a live demo feed.

I wish we were able to communicate better with the design for this prototype but I am happy that it was able to work on an Ipad.

MY JOURNEY SO FAR...

I am very progressive when it comes to building websites and I strive to use better techniques each time I create.

My interest for web design began in high school where I would constantly customize online blogs and myspace sites. I've always had a natural eye for noticing creative visual communication which has led me to discover my skills in art design through a variety of techniques.

I studied web design at the Art Institute of Orange County in CA., where I learned how to implement hierarchy, typography, grids, imagery, and wireframes, for creating company branding, e-commerce sites, thoughtful user experience & interaction, and functional interfaces. These skills led me to my first Front-End Developer job at Insteo, a digital signage company.

At Insteo, I worked as the main developer for creating their customizable web applications. I created a variety of interactive websites that were used at events, restaurants, hotels and stores.

I am very progressive when it comes to building websites and I strive to use better techniques each time I create.

I decided to pursue my dreams further and expand my knowledge in the developing side of websites. I attended the Coding Dojo in March of 2015 where I solidified my knowledge in creating an entire website from front to back.

The Coding Dojo is a 12 Week Bootcamp for learning 3 full stacks of web development. It is one of the best decisions I've ever made for myself. Weeks of morning algorithms and learning new technologies has broadened my idea of what it takes to fully create a website application.

WHO I'VE BECOME

My experience in learning web development has given me the ability to see the bigger picture when I create a website. Previously my coding world was html, css, javascript, jquery, php and mysql, but now I can view them as parts of a LAMP stack application. I have more knowledge of different web frameworks and how they can be implemented to jumpstart an application. I can deliver you a website quickly on Ruby on Rails, give you a site built in the MEAN stack which can offer you a more real-time experience with the use of socket.io, or I can research and learn a new framework that might be more fitting for your application. Frameworks not only for back end development, but front end such as Bootstrap or Materialize can be used to help achieve a more unified design and have it work responsively.

I have a greater understanding of how MVC's work and why they exist.

I am thankful for things like GitHub, StackOverflow, CodePen, JSBin, NPM, and any number of frameworks and libraries you can find. It has given me much more appreciation for programmers, designers, and the field of web. I program with a better mindset for making my code understandable, maintainable for others, and follow customary web techniques.

Because of my new knowledge, I have started to program with functionality first and then rework the design afterwards. Using a more methodical approach, we can build the basic functionalities of a web application and see the pieces we are able to use for design then create the right experience to generate more attraction.

I have grown as a designer to believe in the idea of practicality and ease; for me, this idea serves as a basis for creating a functional web experience that serves the user and requires user interaction only when completely necessary but must also be completely easy. I also love minimalism and high resolution images, especially free ones. I also think there's a certain charm to flat design but because it's becoming a trend, at some point you would need to consider an update. I've learned to sacrifice for time constraints and not to be too afraid to replicate. Familiarity builds trust and can strengthen the overall user experience.

Practicality and ease is also reflected through my programming. We shouldn't always feel compelled to reinvent the wheel, but we should work together and use the great wheels others have created as a foundation for a brand new product. All of these ideas have driven me to become a much more refined and resourceful web developer.

LOOKING FORWARD

I look at my future with a big grin. Excited to meet and work with other programmers, designers, and directors. Eager to discover what new trends and technologies I should know and learn. My appetite for learning new languages has grown. I see myself bridging the gap between the designer and the programmer. I look forward to transcending my skills within a company and get a feel about the business side of web development. I am ready to further my career and some day become Chief Technology Officer of a big company.
